# Issue Memory

This repo keeps issue memory for repeatable harness, workflow, tooling, or docs-governance failures.

## Capture rules

- Record repeatable failures, not one-off confusion.
- Prefer operational triggers first and wording cleanups second.
- Do not seed fake incidents.
- Use issue memory to support future docs sync or bootstrap maintenance decisions.

## Current state

No issue-memory entries are seeded during bootstrap adoption. Add entries only after a real repeated issue occurs.
