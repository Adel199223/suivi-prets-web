# flutter_app

Use this archetype for Flutter apps that may target mobile, desktop, or web.

## Good fit
- cross-platform reader tools
- apps with shared Dart logic across platforms
- repos where `flutter doctor`, run, and build guidance should be first-class

## Default posture
- mode: `standard`
- keep commands explicit and copy-paste friendly
- include beginner support when the operator is a non-coder

## Recommended modules
- `beginner_support`
- `diagnostics`
- `build_identity`
- `qa_checks`
- `local_env_overlay`
