# api_service

Use this archetype for backend services, APIs, or integration-heavy workers.

## Good fit
- repos with deployment, environment, or service integration concerns
- apps that need stronger release and roadmap structure

## Default posture
- mode: `full`
- host integration and release checklists matter more than in lightweight repos

## Recommended modules
- `host_integration`
- `release_management`
- `roadmap_governance`
- `evaluation`
